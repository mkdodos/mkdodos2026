加入報價列印功能,同時也有將原本薪資PDF的字型檔移出src,程式以不同方式處理
為求慎,在更新前, package.json 做修改
`"homepage": "/echoway2026/build"`
接著執行
`npm run build`
做完後,將 build 資料夾 放到 htdocs/echoway2026
測試點選報價頁面正常(http://192.168.0.10:8888/echoway2026/build/quotation)
但按下列印,到此頁面(http://192.168.0.10:8888/echoway2026/build/quotation-pdf)
出現 Error 404
開發時,列印頁面正常
(http://localhost:3003/echoway2026/build/quotation-pdf)

App.js 中的路由如下
`<Route path="/quotation-pdf" element={<QuotationPdf />} />`

列印鈕有用到
`window.open(`quotation-pdf`, "_blank");`

這種開啟新分頁的行為，會直接向 Apache 伺服器發送請求。
Apache 收到 `http://192.168.0.10:8888/echoway2026/build/quotation-pdf` 請求時，會在硬體空間尋找名為 `quotation-pdf` 的資料夾或檔案。因為根本沒有這個實體檔案，Apache 就直接回傳 404 錯誤。

但是當你打包成靜態檔案（`npm run build`）並放到 Apache 的 `htdocs` 後，**重新整理頁面或直接輸入網址時，Apache 會以為你要在 `echoway2026/build/quotation-pdf` 資料夾下尋找一個實體的檔案（例如 `index.html`）**。因為找不到這個實體檔案，Apache 就直接回傳了 **Error 404**。

解決方式

在伺服器的 `htdocs/echoway2026/build/` 資料夾內，新增一個 `.htaccess` 檔案，告訴 apache
如果請求的不是實體檔案或資料夾，一律回傳 `index.html`，這樣就可讓 React Router 自己處理路由。

```
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /echoway2026/build/
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /echoway2026/build/index.html [L]
</IfModule>
```
