**檢查 IIS 網站日誌（IIS Logs）**

- 預設路徑在：`C:\inetpub\logs\LogFiles\`


欄位順序
#Fields: date time s-ip cs-method cs-uri-stem cs-uri-query s-port cs-username c-ip cs(User-Agent) cs(Referer) sc-status sc-substatus sc-win32-status time-taken


最後一個欄位叫做 **`time-taken`**。 在 IIS 中，`time-taken` 的預設單位是**毫秒（ms，Milliseconds）**。它代表的意思是：_從 IIS 接收到請求的第一個位元組開始，到 IIS 將最後一個回應位元組傳送完畢並關閉連線為止，總共花費的時間。_


| **日誌中的位置** | **對應欄位名稱**       | **實際範例數據**                        | **說明**             |
| ---------- | ---------------- | --------------------------------- | ------------------ |
| 第 1 欄      | `date`           | 2026-07-02                        | 日期                 |
| 第 2 欄      | `time`           | 01:12:56                          | 時間（預設為 UTC 時間）     |
| 第 5 欄      | `cs-uri-stem`    | `/erpdev/Delivery/car_route.aspx` | **被請求的網頁路徑**       |
| 第 11 欄     | `sc-status`      | `200`                             | HTTP 狀態碼（200 代表成功） |
| **最後一欄**   | **`time-taken`** | **`678`** 或 **`5820`**            | **處理這張網頁所花費的毫秒數**  |




但可以看到某些特定頁面（如 `/erpdev/works/workDetailsDevExpress.aspx`）花費了高達 **`5820 ms`**（將近 6 秒）才回應 。


2026-07-02 01:14:05 192.168.1.6 GET /erpdev/works/workDetailsDevExpress.aspx - 80 - 192.168.1.29 ... http://... 200 0 0 5820

$$\text{5820 毫秒} \div \text{1000} = \text{5.82 秒（將近 6 秒）}$$


### 結合 `200 0 0 5820` 的最終診斷結論

把整段日誌的末端連起來看：

`200`（成功） $\rightarrow$ `0`（無子錯誤） $\rightarrow$ `0`（底層網路與系統無中斷） $\rightarrow$ `5820`（花了 5.8 秒）
